# Productboard Search

Use this guidance-only skill to improve search workflows in Productboard. It does not call external tools directly.

## When to use

- You need faster retrieval of feature and feedback context.
- You want to reduce duplicate requests and fragmented discovery.
- You need consistent search patterns across PM teams.

## Core workflows

### 1) Search conventions

- Define naming standards for features and initiatives.
- Use structured keywords for customer pain, segment, and outcome.
- Keep aliases for common terms used by different teams.

### 2) Discovery workflow

- Start with broad terms, then narrow by product area and status.
- Validate whether similar requests already exist before creating new items.
- Capture evidence links and user context for each discovered item.

### 3) Review and deduplication

- Run weekly duplicate checks for top request clusters.
- Merge overlapping items with explicit rationale.
- Keep a "canonical item" policy for each recurring theme.

## Risk controls

- Avoid creating parallel feature records for the same outcome.
- Prevent false conclusions from isolated feedback samples.
- Keep search tags and status definitions synchronized across teams.

## Output format

When asked for help, provide:

- A search playbook with query patterns.
- A deduplication checklist.
- A template for evidence-backed feature discovery notes.
