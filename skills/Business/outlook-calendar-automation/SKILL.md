# Outlook Calendar Operations

Use this guidance-only skill to improve scheduling and calendar operations. It does not execute API calls or scripts.

## When to use

- You need structured scheduling policies across teams.
- You want fewer low-value meetings and clearer outcomes.
- You need stronger follow-up from meetings and commitments.

## Core workflows

### 1) Scheduling standards

- Define default meeting lengths, buffers, and focus-time protection.
- Apply meeting-type rules (decision, status, planning, retrospective).
- Require minimum scheduling metadata (purpose, owner, attendees, pre-read).

### 2) Availability and capacity planning

- Establish no-meeting blocks and deep-work windows.
- Track overloaded calendars and rebalance meeting ownership.
- Use attendee tiers (required vs optional) to reduce unnecessary load.

### 3) Meeting quality and follow-up

- Require agenda and expected decisions before acceptance.
- Capture decisions, owners, and due dates at meeting close.
- Run weekly follow-up audits for overdue action items.

### 4) Executive and cross-team coordination

- Standardize stakeholder update cadences and prep checklists.
- Use recurring review slots for risk and dependency management.
- Keep a lightweight escalation path for urgent scheduling conflicts.

## Risk controls

- Avoid recurring meetings without measurable value.
- Prevent action-item loss with explicit owner and deadline.
- Limit urgent reschedules that create planning instability.

## Output format

When asked for help, provide:

- A scheduling governance policy.
- A meeting quality checklist.
- A follow-up tracker template with SLA rules.
