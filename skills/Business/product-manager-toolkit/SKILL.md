# Product Manager Toolkit

Use this guidance-only skill to structure end-to-end product management workflows. It does not execute scripts or external tools.

## When to use

- You need a repeatable prioritization process.
- You want stronger discovery-to-delivery handoffs.
- You need consistent product documentation quality.

## Core workflows

### 1) Prioritization

- Use a shared scoring model (impact, confidence, effort, risk).
- Evaluate strategic alignment before committing roadmap capacity.
- Log trade-offs and explicit "not now" decisions.

### 2) Discovery synthesis

- Consolidate research themes by user segment and job-to-be-done.
- Separate observed problems from proposed solutions.
- Validate top assumptions with lightweight experiments.

### 3) PRD and handoff quality

- Require problem statement, success metrics, scope, and non-goals.
- Define acceptance criteria and edge cases clearly.
- Include launch and rollback considerations in delivery planning.

### 4) Go-to-market coordination

- Align product, marketing, sales, and support on launch narrative.
- Define enablement artifacts and ownership per function.
- Run post-launch reviews against expected outcomes.

## Risk controls

- Avoid output-heavy planning without decision quality.
- Prevent metric ambiguity by using metric dictionaries.
- Limit roadmap churn with change-control checkpoints.

## Output format

When asked for help, provide:

- A prioritization table template.
- A PRD quality rubric.
- A launch readiness checklist.
