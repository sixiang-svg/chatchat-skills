# Todoist Task Operations

Use this guidance-only skill to structure task management practices in Todoist. It does not execute MCP or API actions.

## When to use

- You need a standardized task structure across workstreams.
- You want better prioritization and due-date discipline.
- You need weekly review routines that drive completion.

## Core workflows

### 1) Project and section design

- Define project hierarchy by outcome area, not tool/team alone.
- Use sections for intake, in-progress, blocked, and done states.
- Keep naming conventions and ownership clear.

### 2) Prioritization and scheduling

- Define priority rules (impact, urgency, dependency, effort).
- Limit same-day overcommitment with capacity boundaries.
- Use recurring tasks for operational routines and controls.

### 3) Review and accountability

- Run daily triage and weekly planning reviews.
- Resolve stale tasks and overdue items with explicit next actions.
- Track completion reliability by owner and project.

## Risk controls

- Avoid bloated backlogs without archive rules.
- Prevent unowned tasks by enforcing assignment standards.
- Keep due dates realistic and linked to dependencies.

## Output format

When asked for help, provide:

- A Todoist project taxonomy.
- A prioritization policy.
- A weekly review checklist.
