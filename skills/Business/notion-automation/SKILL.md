# Notion Workspace Operations

Use this guidance-only skill to design Notion workflows and governance. It does not execute external tools or scripts.

## When to use

- You need a scalable docs and database structure in Notion.
- You want stronger ownership and freshness for operational pages.
- You need repeatable templates for projects, decisions, and runbooks.

## Core workflows

### 1) Information architecture

- Separate knowledge bases by function (product, operations, sales, support).
- Define parent-child page hierarchy and archive rules.
- Keep a single source of truth for policies and SOPs.

### 2) Database and template governance

- Standardize key fields (owner, status, due date, priority, last reviewed).
- Build templates for recurring artifacts (PRDs, postmortems, meeting notes).
- Enforce naming conventions and required fields.

### 3) Review and maintenance cadence

- Run monthly stale-page reviews and quarterly structure audits.
- Track pages without owners and resolve orphaned content quickly.
- Set reminders for policy and compliance document refresh.

### 4) Collaboration and decision logging

- Capture decisions with rationale, date, and accountable owner.
- Link decisions to projects and follow-up actions.
- Publish concise weekly summaries of notable changes.

## Risk controls

- Prevent duplicate documentation across teams.
- Avoid undocumented decision changes in critical processes.
- Keep sensitive pages access-scoped and regularly audited.

## Output format

When asked for help, provide:

- A workspace architecture map.
- A database schema and template checklist.
- A documentation hygiene and governance plan.
