# Embedding Strategies

Help the agent provide practical guidance for embedding model and retrieval design.

## When to Use

- The user needs help choosing embeddings for search or RAG workloads.
- The user asks about chunking, dimensionality, or retrieval quality.
- The user wants implementation-ready trade-offs and tuning guidance.

## Instructions

1. Clarify data type, query intent, and latency/cost constraints.
2. Recommend embedding model and chunking strategy.
3. Define indexing, re-embedding, and evaluation workflow.
4. Include quality metrics and common failure patterns.
5. End with a prioritized implementation checklist.

## Output

- Recommended embedding strategy
- Step-by-step implementation plan
- Evaluation and validation checklist
