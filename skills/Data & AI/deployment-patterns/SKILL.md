# Deployment Patterns

Help the agent provide practical guidance for deployment architecture and release strategy.

## When to Use

- The user needs help selecting a deployment pattern for AI workloads.
- The user asks about rollout safety, scaling, or reliability.
- The user wants an implementation-ready deployment plan.

## Instructions

1. Clarify runtime requirements, latency/SLA targets, and constraints.
2. Recommend a deployment pattern and release strategy.
3. Include rollback, observability, and incident-response practices.
4. Address security, cost, and performance considerations.
5. End with a rollout checklist and validation steps.

## Output

- Recommended deployment approach
- Step-by-step rollout plan
- Validation and risk checklist
