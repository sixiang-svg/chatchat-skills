# Database Migrations Migration Observability

Support database migrations migration observability workflows with clear steps and best practices.

## Instruction
- Identify the source and target database environments and determine the specific migration strategy, such as zero-downtime or refactor.
- Establish performance baselines by collecting pre-migration metrics like query latency and resource utilization.
- Configure real-time monitoring to track critical migration data, including replication lag and error rates.
- Define health check parameters and automated alerts to detect anomalies during the data transfer phase.
- Implement post-migration data integrity verification to ensure consistency between the source and target datasets.
- Prepare a rollback monitoring plan to maintain observability in case the migration needs to be reversed.

## When to Use
- When planning or executing database upgrades, cloud transitions, or platform migrations.
- When managing zero-downtime migrations that require precise synchronization monitoring.
- When auditing the success and performance impact of a completed migration project.

## Output
- A comprehensive migration observability plan with defined metrics and thresholds.
- Real-time status summaries and risk assessments during the active migration phase.
- Post-migration health reports and data integrity verification findings.
