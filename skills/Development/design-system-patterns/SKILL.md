# Design System Patterns

Design reusable UI patterns that maintain visual and behavioral consistency across products.

## When to Use

- You are building or extending a component library/design system.
- Product UI is inconsistent across teams or modules.

## Workflow

1. Identify shared UI primitives and interaction patterns.
2. Map components to tokenized spacing, typography, color, and states.
3. Define variant APIs and composition boundaries.
4. Enforce accessibility and responsive behavior standards.
5. Provide migration steps for adopting teams.

## Output

- Pattern and component architecture
- Token and variant conventions
- Adoption and governance checklist
