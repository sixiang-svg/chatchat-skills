# Avalonia Zafiro Development

Guide feature development in Avalonia apps using predictable architecture and implementation patterns.

## When to Use

- You are building or extending features in an Avalonia desktop app.
- You need consistency across screens, state, and interaction flows.

## Workflow

1. Break feature requirements into views, viewmodels, and services.
2. Define navigation, bindings, and validation behavior for each screen.
3. Recommend folder/module layout that fits existing project conventions.
4. Identify reusable components and shared interaction patterns.
5. Add tests and quality checks for critical UI behavior.

## Output

- Feature implementation plan
- Suggested Avalonia/Zafiro module structure
- Risk notes and verification checklist
