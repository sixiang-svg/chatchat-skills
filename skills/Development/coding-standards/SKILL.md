# Coding Standards

Create enforceable coding standards that teams can use during implementation and review.

## When to Use

- You need a shared coding standard for a project or team.
- Existing code quality is inconsistent across contributors.

## Workflow

1. Identify language/framework-specific style and architecture conventions.
2. Define rules for naming, structure, error handling, and tests.
3. Add examples of compliant and non-compliant code patterns.
4. Align standards to tooling (linters, formatters, CI checks).
5. Produce rollout guidance for incremental adoption.

## Output

- Standards document outline
- Enforcement and tooling checklist
- Migration notes for legacy code
