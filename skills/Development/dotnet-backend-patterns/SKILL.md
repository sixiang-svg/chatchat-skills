# .NET Backend Patterns

Provide pattern-level guidance for maintainable and scalable .NET backend systems.

## When to Use

- You are implementing backend features in ASP.NET/.NET.
- You need consistency for service design, data access, and resilience.

## Workflow

1. Map domain boundaries and service responsibilities.
2. Recommend API, application, and infrastructure layering.
3. Define validation, error handling, and transaction patterns.
4. Add resilience controls for external dependencies.
5. Provide testing and observability guidance.

## Output

- .NET architecture and pattern guide
- Implementation recommendations
- Reliability and test checklist
