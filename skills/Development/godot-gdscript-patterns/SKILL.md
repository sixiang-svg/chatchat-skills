# Godot GDScript Patterns

Provide practical GDScript guidance that improves readability and game-system reliability.

## When to Use

- You are building or refactoring Godot gameplay code.
- You need better structure for scene logic and communication.

## Workflow

1. Identify scene responsibilities and node ownership.
2. Define state and signal boundaries across scripts.
3. Apply reusable patterns for input, movement, and events.
4. Reduce tight coupling between scenes/components.
5. Add test/debug hooks and performance checks.

## Output

- GDScript architecture recommendations
- Scene/signal pattern map
- Refactor and validation checklist
