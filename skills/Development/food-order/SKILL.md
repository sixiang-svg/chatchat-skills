# Food Order

Implement reliable food-ordering features across UI, API, and operational edge cases.

## When to Use

- You are building a food-ordering product or feature.
- You need structured guidance for order, payment, and delivery logic.

## Workflow

1. Define order lifecycle states and allowed transitions.
2. Model cart, item customization, discounts, tax, and fees.
3. Implement checkout validation and payment error handling.
4. Add fulfillment integration, status updates, and user notifications.
5. Provide test scenarios for high-risk edge cases (stock, retries, cancellations).

## Output

- Food-order feature architecture
- Validation and reliability rules
- End-to-end test checklist
