# Mobile Design

Create mobile design plans that are user-centered and practical for engineering implementation.

## When to Use

- You are defining UX for new or existing mobile features.
- You need design guidance aligned with platform conventions.

## Workflow

1. Define user tasks, constraints, and mobile context.
2. Map flows and screen hierarchy with iOS/Android conventions.
3. Specify interaction states (loading, error, offline, success).
4. Ensure accessibility and touch ergonomics.
5. Hand off implementation-ready specs and QA criteria.

## Output

- Mobile UX flow and screen plan
- Interaction and accessibility guidelines
- Design QA checklist
