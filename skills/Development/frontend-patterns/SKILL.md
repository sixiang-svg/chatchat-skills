# Frontend Patterns

Apply proven frontend patterns that keep interfaces maintainable as product complexity grows.

## When to Use

- You are designing or refactoring frontend feature architecture.
- You need guidance on patterns for state, composition, and interactions.

## Workflow

1. Analyze feature complexity, state boundaries, and data dependencies.
2. Select suitable patterns for components and state orchestration.
3. Define communication contracts between UI layers.
4. Add error/loading/empty-state behavior patterns.
5. Provide testing and performance validation strategy.

## Output

- Pattern recommendation and rationale
- Refactor/implementation blueprint
- Quality and performance checklist
