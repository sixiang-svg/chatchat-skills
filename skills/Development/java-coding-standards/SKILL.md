# Java Coding Standards

Establish practical Java conventions that are enforceable in PR reviews and tooling.

## When to Use

- Your Java codebase needs consistent standards across contributors.
- You want clearer guidance for reviews, refactors, and new features.

## Workflow

1. Define package, class, and method-level style conventions.
2. Set standards for architecture layering and dependency direction.
3. Establish exception, logging, and nullability handling rules.
4. Add testing expectations for unit/integration coverage.
5. Map standards to linters/formatters and review checklists.

## Output

- Java standards specification
- Tooling and enforcement recommendations
- Migration plan for existing code
